# STL 型腔收纳与旧收纳盒迁移需求文档

更新时间：2026-03-15

这份文档用于明确 STL 相关的两个目标工作流，避免继续把“实体改底”“标准矩形规整”“型腔收纳”“旧收纳盒迁移”混在同一个模板语义里。

这是一份需求文档，不是当前代码实现说明。当前仓库中的 `stl-retrofit` 只会把导入 STL 的包围盒规整成标准矩形 Gridfinity 实体，它不满足本文定义的两个目标工作流。

## 1. 背景与问题

当前讨论里实际上存在两类完全不同的需求：

1. 把一个“被收纳物”的 STL 直接转换为新的 Gridfinity 型腔收纳盒
2. 把一个“旧收纳盒/旧托盘”的 STL 中已有的负形提取出来，再迁移成新的 Gridfinity 型腔收纳盒

这两类需求最终产物都应该是：

- 外轮廓为标准矩形的 Gridfinity 盒体
- 内部存在用于收纳的型腔

但它们的输入语义不同：

- 第一类输入的是“物品本体”
- 第二类输入的是“已有收纳盒”

如果继续把它们塞进同一个“STL 改底适配”模板，会持续产生歧义。

## 2. 术语定义

- `被收纳物`
  - 指真正需要被放进盒子里的物体
- `正形`
  - 物体本身的实体几何
- `负形 / 型腔`
  - 用于容纳物体的空腔几何
- `旧收纳盒`
  - 已存在的非 Gridfinity 收纳托盘、收纳盒、夹具或展示底座
- `迁移`
  - 从旧收纳盒中提取已有型腔语义，并重建为新的标准 Gridfinity 盒体

## 3. 目标工作流总览

建议拆成两个独立模板，而不是继续复用当前 `stl-retrofit`：

### 3.1 工作流 A：STL 型腔收纳

建议模板：

- 模板 id：`stl-cavity-bin`
- 中文名：`STL 型腔收纳`

输入：

- 一个代表“被收纳物”的 STL

输出：

- 一个外轮廓为标准矩形的 Gridfinity 收纳盒
- 盒体内部挖出与该 STL 对应的型腔

### 3.2 工作流 B：旧收纳盒迁移

建议模板：

- 模板 id：`legacy-organizer-migrate`
- 中文名：`旧收纳盒迁移`

输入：

- 一个代表“旧收纳盒/旧托盘”的 STL

输出：

- 一个外轮廓为标准矩形的 Gridfinity 收纳盒
- 盒体内部型腔来自旧收纳盒中被提取出的已有负形

### 3.3 当前 `stl-retrofit` 的定位

当前 `stl-retrofit` 如果保留，建议明确改名为类似：

- `STL 实体规整`
- `实体改底适配`

它的语义应该与本文两个型腔工作流彻底分开。

## 4. 共同产品要求

无论是工作流 A 还是 B，都应满足以下共性要求：

- 最终输出是标准矩形 Gridfinity 外轮廓
- 最终模型是“可收纳”的盒体，而不是纯实心块
- 自动推荐最小可用 `gridX / gridY / heightUnits`
- 允许切换到固定外部尺寸
- 支持 `ASCII STL` 与 `Binary STL`
- 支持页面内 `X / Y / Z` 的 `90°` 步进旋转
- 支持 3D 预览
- 支持导出 STL
- 支持磁铁孔等已有 Gridfinity 通用选项的复用

共同非目标：

- 不支持任意角度旋转
- 不支持多个 STL 一次性合并
- 不支持自动缩放 STL 到目标尺寸
- 不承诺修复严重非流形、开口曲面、坏法线模型
- 首版不追求复杂多腔、多层嵌套、布料类可变形物体

## 5. 工作流 A 详细规格：STL 型腔收纳

## 5.1 模板定位

这个模板解决的是：

- 用户手里有一个物品的 STL
- 用户想要一个新的 Gridfinity 标准收纳盒
- 系统应该基于该物品 STL 直接生成型腔

典型输入：

- 工具
- 零件
- 玩具部件
- 电子配件
- 小夹具

## 5.2 用户流程

固定流程建议为：

1. 进入 `STL 型腔收纳`
2. 上传物品 STL
3. 在页面中旋转物品朝向
4. 预览物品包围盒尺寸
5. 选择 `自动推荐` 或 `固定尺寸`
6. 设置型腔相关参数
7. 生成标准 Gridfinity 盒体
8. 在盒体内部挖出型腔
9. 预览并导出 STL

## 5.3 首版参数建议

首版建议至少包含：

- `rotationX / rotationY / rotationZ`
- `sizeMode`
- `gridX / gridY / heightUnits`
- `xyClearance`
- `zClearance`
- `floorThickness`
- `wallThickness`
- `magnetHoles`

可选但不强制首版完成：

- `gripCutout`
- `labelLip`
- `lockOrientation`

## 5.4 几何语义

首版几何流程应为：

1. 解析并缓存物品 STL
2. 按 `90°` 步进旋转求得当前朝向
3. 基于物品包围盒和清隙参数计算所需内腔尺寸
4. 自动搜索最小可用 `gridX / gridY / heightUnits`
5. 生成标准矩形 Gridfinity 盒体
6. 将物品几何按指定清隙扩张后，从盒体中减去，形成负形型腔

重要要求：

- 物品 STL 必须参与最终布尔，不能只拿来算包围盒
- 最终结果必须是“标准盒体 + 内部型腔”
- 预览中应该能看出物品轮廓信息仍然存在于型腔中

## 5.5 验收标准

满足以下条件才算通过：

- 不上传 STL 时不给出可导出的结果
- 上传 STL 后，最终模型外轮廓是标准矩形
- 最终模型不是纯实心块
- 最终模型内部型腔与 STL 形状对应，而不是只按包围盒挖矩形槽
- 自动推荐尺寸应足以容纳物品与清隙

## 6. 工作流 B 详细规格：旧收纳盒迁移

## 6.1 模板定位

这个模板解决的是：

- 用户手里没有“物品 STL”
- 用户手里有一个旧收纳盒/旧托盘 STL
- 旧模型里已经存在适配某个物品的负形
- 用户希望把这个负形迁移成新的 Gridfinity 标准收纳盒

本质上，这个模板是：

- 先做“型腔提取”
- 再进入“型腔收纳生成”

## 6.2 与工作流 A 的区别

工作流 A：

- 输入的是物品本体
- 直接把正形转成负形

工作流 B：

- 输入的是旧收纳盒
- 先从旧收纳盒中识别和提取已有负形
- 再把这个负形作为新的型腔来源

因此，B 比 A 多了一步“反求型腔”。

## 6.3 首版边界

为了保证首版可落地，建议先限制在以下条件：

- 单个 STL
- 单个主型腔
- 旧收纳盒在某一朝向上存在清晰的开口方向
- 型腔主要朝上开口，或可通过 `90°` 旋转摆正到朝上开口
- 不支持复杂多腔、多层抽屉、完全封闭内腔
- 不支持自动理解软质物体、卡扣变形结构

如果提取失败，应明确失败，不应静默退化成纯矩形块。

## 6.4 用户流程

建议流程：

1. 进入 `旧收纳盒迁移`
2. 上传旧收纳盒 STL
3. 通过 `X / Y / Z` 旋转把开口方向摆正
4. 系统尝试提取主要型腔
5. 向用户展示“源模型 / 提取出的型腔 / 最终 Gridfinity 结果”
6. 用户确认提取结果是否正确
7. 系统基于提取型腔生成新的标准 Gridfinity 收纳盒
8. 预览并导出 STL

## 6.5 型腔提取要求

首版不要求做到通用 CAD 逆向，但至少应支持：

- 从一个开口托盘类模型中识别主要空腔
- 提取一个可用于后续布尔的闭合或可稳定求解的型腔代理几何
- 允许用户在提取后做简单确认，而不是黑盒直接导出

首版可以接受的简化：

- 只提取最大主腔
- 只支持自上而下开口
- 必要时先把源模型包在一个辅助包围体里，再通过布尔求得空腔区域

## 6.6 验收标准

满足以下条件才算通过：

- 对简单旧托盘 STL，系统能提取出主要负形
- 提取失败时给出明确提示，而不是产出无意义矩形块
- 最终输出仍是标准矩形 Gridfinity 盒体
- 最终型腔应明显继承旧收纳盒的内部形状，而不只是源模型外轮廓包围盒

## 7. UI 与交互要求

建议不要继续把所有 STL 相关能力塞进一个模板。

推荐做法：

- 首页提供两个独立模板入口
- 两个模板共用 STL 导入组件和旋转组件
- 但拥有各自独立的信息面板和说明文案

工作流 A 重点展示：

- 物品尺寸
- 当前朝向
- 推荐外部尺寸
- 型腔清隙

工作流 B 重点展示：

- 旧收纳盒尺寸
- 当前开口朝向
- 提取到的型腔状态
- 提取成功/失败原因

## 8. 数据结构与架构方向

建议的数据层拆分：

- `ImportedStlSourceSummary`
  - 继续复用当前 STL 摘要结构
- `StlCavityBinParams`
  - 工作流 A 参数
- `LegacyOrganizerMigrateParams`
  - 工作流 B 参数
- `ExtractedCavitySummary`
  - 工作流 B 的提取结果摘要

worker 层建议新增：

- `import-stl`
  - 继续沿用当前导入逻辑
- `extract-cavity`
  - 旧收纳盒迁移专用

生成层建议拆成：

- `stlCavityBin.ts`
- `legacyOrganizerMigrate.ts`

而不是继续往 `stlRetrofit.ts` 里叠语义。

## 9. 推荐开发顺序

建议分两阶段推进：

### 阶段 1：STL 型腔收纳

先做工作流 A，因为它更直接：

- 输入语义清晰
- 几何路径更短
- 能先沉淀 STL 型腔布尔与自动尺寸推荐能力

### 阶段 2：旧收纳盒迁移

再做工作流 B，因为它依赖阶段 1 的能力，并新增：

- 型腔提取
- 提取结果审查
- 失败态与边界约束

## 10. 对当前代码的明确结论

当前 `stl-retrofit` V1：

- 会导入 STL
- 会计算尺寸
- 会生成标准矩形 Gridfinity 实体
- 不会保留原物体几何
- 不会把原物体转成型腔
- 也不会从旧收纳盒提取已有负形

因此：

- 它不满足工作流 A
- 也不满足工作流 B

后续开发不应继续把它解释成“型腔收纳”或“旧收纳盒迁移”。
